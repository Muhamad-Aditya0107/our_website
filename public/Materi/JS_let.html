JavaScript let

Kata kunci let diperkenalkan dalam ES6 (2015) dan digunakan untuk mendeklarasikan variabel dengan cakupan blok (block scope).

Properti Variabel let

  • Variabel yang dideklarasikan dengan let memiliki cakupan blok.
  • Variabel yang dideklarasikan dengan let harus dideklarasikan sebelum digunakan.
  • Variabel yang dideklarasikan dengan let tidak dapat dideklarasikan ulang dalam cakupan yang sama.

Cakupan Blok (Block Scope)

Sebelum ES6 (2015), JavaScript hanya memiliki cakupan global dan cakupan fungsi. ES6 memperkenalkan kata kunci let dan const, yang memberikan cakupan blok.

Contoh:

{
 let x = 2; 
}
// x tidak dapat digunakan di sini

Cakupan Global dengan var

Variabel yang dideklarasikan dengan var selalu memiliki cakupan global, bahkan di dalam blok.

Contoh:

{
 var x = 2; 
}
// x dapat digunakan di sini

Tidak Dapat Dideklarasikan Ulang

Variabel yang dideklarasikan dengan let tidak dapat dideklarasikan ulang dalam cakupan yang sama:

let x = "John Doe";
let x = 0; // Error

Sebaliknya, variabel yang dideklarasikan dengan var dapat dideklarasikan ulang:

var x = "John Doe";
var x = 0; // Tidak ada error

Perbedaan dalam Deklarasi Ulang

Ketika mendeklarasikan ulang variabel dengan var, variabel di luar blok juga akan terpengaruh:

var x = 10;
// Di sini x adalah 10
{
 var x = 2;
// Di sini x adalah 2
}
// Di sini x juga 2

Namun, dengan let, variabel di luar blok tidak terpengaruh:

let x = 10;
// Di sini x adalah 10
{
 let x = 2;
// Di sini x adalah 2
}
// Di sini x tetap 10

Perbandingan var, let, dan const

Fitur var let const
Cakupan Global Blok Blok
Dapat Dideklarasikan Ulang Ya Tidak Tidak
Dapat Diubah Nilainya Ya Ya Tidak

Kesimpulan

Gunakan let untuk mendeklarasikan variabel yang nilainya dapat berubah, tetapi hanya berlaku dalam cakupan blok. Hal ini membantu menghindari masalah cakupan yang sering terjadi saat menggunakan var.
