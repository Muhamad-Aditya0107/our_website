JavaScript Strict Mode

Strict Mode adalah fitur di JavaScript yang memperketat aturan dalam penulisan kode. Mode ini membantu mencegah kesalahan umum, meningkatkan keamanan, dan membuat kode lebih mudah untuk dioptimalkan oleh mesin JavaScript.

Cara Mengaktifkan Strict Mode
  • Global Scope
    • Tambahkan "use strict"; di awal file atau skrip.
    "use strict";
    x = 10; // Error: x is not defined
  • Fungsi
    • Tambahkan "use strict"; di dalam fungsi untuk mengaktifkan mode hanya pada fungsi tersebut.
    function myFunction() {
    "use strict";
    y = 10; // Error: y is not defined
    }
Perubahan Perilaku di Strict Mode
  • Variabel Harus Dideklarasikan
    • Tidak boleh menggunakan variabel tanpa deklarasi (var, let, atau const).
    "use strict";
    x = 10; // Error: x is not defined
  • Mencegah Duplikasi Properti atau Parameter
    • Tidak boleh ada properti objek atau parameter fungsi dengan nama yang sama.
    "use strict";
    const obj = {
    name: "John",
    name: "Doe" // Error: Duplicate property
    };
  • Menangkap Kesalahan Penulisan
    • Strict mode menangkap kesalahan yang biasanya diabaikan.
    "use strict";
    let public = 10; // Error: Menggunakan nama kata kunci
  • Fungsi Tanpa this
    • Nilai this dalam fungsi biasa akan menjadi undefined, bukan window (di luar strict mode, this akan menjadi objek global window).
    "use strict";
    function test() {
    console.log(this); // Output: undefined
    }
    test();
  • Mencegah Penambahan Properti ke Objek Non-Ekstensibel
    • Tidak bisa menambah properti ke objek yang tidak dapat diperluas.
    "use strict";
    Object.preventExtensions(obj);
    obj.newProp = "test"; // Error
  • Mencegah Penghapusan Properti Tak Dapat Dihapus
    • Properti seperti Object.prototype tidak dapat dihapus.
    "use strict";
    delete Object.prototype; // Error
Kelebihan Strict Mode
  • Meningkatkan Keamanan Kode Membantu mencegah bug karena aturan yang lebih ketat.
  • Membantu Debugging Memberikan error di tempat yang sebelumnya hanya dilewati.
  • Meningkatkan Performa Browser dapat mengoptimalkan kode yang mengikuti strict mode.
  • Memastikan Kompatibilitas Masa Depan Membuat kode lebih kompatibel dengan standar JavaScript modern.
Kesimpulan
  • Strict Mode memperbaiki beberapa kelemahan JavaScript lama.
  • Aktifkan strict mode dengan "use strict";.
  • Berguna untuk debugging, performa, dan keamanan.
